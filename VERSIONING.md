# 버전 관리 규칙

## 표시 형식

앱 하단에는 `버전 1.01 (2026.07.20 00:25)` 형식으로 표시합니다.

- 날짜와 시간은 해당 버전을 확정한 시각입니다.
- 시간대는 대한민국 표준시(`Asia/Seoul`)이며 24시간제로 기록합니다.
- 모든 페이지는 `site-meta.js`의 동일한 버전 정보를 사용합니다.

## 버전 증가 기준

- 작은 수정: 새 블록이나 주요 기능을 추가하지 않는 문구, 데이터, 스타일, 오류 수정은 소수점 둘째 자리를 올립니다. 예: `1.01` → `1.02`.
- 큰 변화: 새 블록·페이지·주요 기능을 추가하거나 화면 구조를 크게 바꾸면 소수점 첫째 자리를 올리고 둘째 자리를 초기화합니다. 예: `1.09` → `1.10`.
- 학년도 변경: 새 학년도용 데이터와 전형 체계로 전환할 때 큰 버전인 정수부를 올리고 소수부를 초기화합니다. 예: `1.xx` → `2.00`.

## 배포 전 확인

1. `site-meta.js`의 `version`과 `releasedAt`을 함께 갱신합니다.
2. 모든 HTML의 `site-meta.js?v=` 값을 같은 버전으로 맞춥니다.
3. 화면 하단의 버전과 날짜·시간을 모바일과 데스크톱에서 확인합니다.
4. 검증이 끝난 변경만 커밋합니다.
